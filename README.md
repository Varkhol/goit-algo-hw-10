**Завдання 1**

Порівняння ефективності жадібного алгоритму та алгоритму динамічного програмування.
Тестування на наборі монет [50, 25, 10, 5, 2, 1].

Час виконання:

| Сума решти | Жадібний алгоритм, сек | Динамічне програмування, сек |
|------------|------------------------|------------------------------|
| 113        | 0.000004               | 0.000051                     |
| 1 000      | 0.000004               | 0.000563                     |
| 10 000     | 0.000003               | 0.004800                     |
| 100 000    | 0.000015               | 0.051326                     |

Час роботи жадібного алгоритму є константним і майже не змінюється із зростанням навантаження. Натомість для динамічного програмування спостерігається пряма лінійна залежність: кожне збільшення суми у 10 разів пропорційно збільшує час виконання теж у 10 разів, що робить цей метод неефективним на великих числах.

Для реальної касової системи жадібний алгоритм є більш оптимальним вибором. Він працює набагато швидше на великих сумах і не забиває оперативну пам'ять масивами.

Жадібний алгоритм може давати неоптимальний результат при нестандартному наборі монет, наприклад [24, 10, 1]. Тому в такому випадку найкраще рішення було б динамічне програмування. Оскільки наш набір монет [50, 25, 10, 5, 2, 1] є канонічним. Це означає, що жадібний алгоритм завжди видає математично найкращий результат - мінімальну кількість монет. Використовувати тут динамічне програмування немає сенсу.


**Завдання 2**
Обчислення інтеграла методом Монте-Карло

Функція
f(x) = x², межі інтегрування: [0, 2]

Результати

| Метод                                | Значення інтеграла     |
|--------------------------------------|------------------------|
| Монте-Карло (100 000 точок)          | 2.649372               |
| Монте-Карло (100 експериментів)      | 2.671795               |
| Аналітичний (x³/3)                   | 2.666667               |
| quad (scipy)                         | 2.666667               |

Висновки
Монте-Карло дає результат близький до аналітичного 8/3 ≈ 2.6667. Серія експериментів точніша за один запуск — працює закон великих чисел.
quad дає точний результат з похибкою ~10⁻¹⁴, Монте-Карло — з похибкою ~10⁻². Для простих функцій quad ефективніший, але Монте-Карло корисний коли аналітичне рішення неможливе.
